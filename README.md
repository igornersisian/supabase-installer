# Supabase Self-Hosted Production Installer\n\n🚀 **Complete production-ready Supabase installation script**\n\n[![Version](https://img.shields.io/badge/version-3.15-blue.svg)](https://github.com/Igor-Nersisyan/supabase-installer)\n[![Ubuntu](https://img.shields.io/badge/Ubuntu-20.04%20|%2022.04%20|%2024.04-orange.svg)](https://ubuntu.com/)\n[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)\n\n## 🎯 What is this?\n\nA battle-tested installer that deploys self-hosted Supabase with enterprise features and critical production fixes.\n\n**Why use this installer?**\nStandard Supabase guides often result in memory leaks, broken Edge Functions (DNS issues), and upload limits. This installer fixes all known issues out of the box.\n\n## ✨ Key Features (v3.15)\n\n### 📦 10GB File Upload Support (Fixed)\n- **Problem solved:** Standard configs use string values for size limits, which causes failures.\n- **Our fix:** Applies correct **Integer** values for `FILE_SIZE_LIMIT` and `UPLOAD_FILE_SIZE_LIMIT`.\n- **Full stack support:** Nginx (11GB), Kong (11GB), and Storage (10GB) are perfectly synced.\n- **TUS Protocol:** Full support for resumable uploads.\n\n### ⚡ Performance & Stability\n- **Memory Optimized:** Analytics container optimized to use ~450MB instead of 1.5GB+.\n- **Edge Functions Stability:** Docker Daemon configured with `dns-opts=[\"ndots:0\"]` to prevent DNS resolution failures after restarts.\n- **Kong Timeouts:** Increased to **5 minutes** (300s) to support long-running AI/n8n workflows (default is 60s).\n\n### 🛡️ Enterprise Security\n- **Database Hardening (v3.3):** Includes a script to whitelist IPs or restrict access to Docker-only.\n- **Auto SSL:** Let's Encrypt certificate with auto-renewal.\n- **Log Rotation:** Docker logs limited to 10MB/file to prevent disk overflow.\n\n## 📋 Requirements\n\n- **OS**: Ubuntu 20.04, 22.04, or 24.04\n- **RAM**: Minimum 3GB (4GB+ recommended)\n- **CPU**: 2+ cores\n- **Storage**: 20GB+ free space\n- **Ports**: 80, 443, 5432 (PostgreSQL)\n\n## 🚀 Quick Install\n\nRun as root:\n\n```bash\n# Download installer\nwget https://raw.githubusercontent.com/Igor-Nersisyan/supabase-installer/main/install.sh\n\n# Make executable\nchmod +x install.sh\n\n# Run\n./install.sh\n```\n\nYou'll be prompted for:\n- Domain name (e.g., `supabase.example.com`)\n- Email address (for SSL)\n\n## 📁 Post-Installation\n\n### 1. Credentials\nAll passwords and API keys are saved in a secured file:\n```bash\ncat /root/supabase-credentials.txt\n```\n*Save these securely and delete the file afterwards.*\n\n### 2. Database Hardening (Critical)\nBy default, PostgreSQL (port 5432) is open. Run the hardening script to secure it:\n\n```bash\nbash /root/harden_supabase_db.sh\n```\n**Options:**\n1. **Same Server:** Restricts access to internal Docker network only (for local n8n).\n2. **Different Server:** Whitelists specific external IPs.\n3. **Add IP:** Add more trusted servers later.\n\n## 🧩 Included Edge Functions\n\nThe installer pre-deploys useful functions:\n\n1.  **`n8n-proxy`**: A public entry point for n8n webhooks.\n    *   URL: `https://your-domain.com/functions/v1/n8n-proxy`\n    *   Requires `N8N_WEBHOOK_URL` in `.env`.\n2.  **`webhook-endpoint-1/2/3`**: Protected endpoints requiring Auth Bearer token.\n3.  **`hello`**: Simple health check function.\n\n## 🛠️ Management\n\n### Service Control\n```bash\ncd /opt/supabase-project\n\n# Check status\ndocker compose ps\n\n# Restart all services\ndocker compose down && docker compose up -d\n\n# Restart only functions (faster)\ndocker compose restart functions\n```\n\n### Checking 10GB Upload Support\nYou can verify the configuration works by checking the container environment:\n```bash\ndocker exec supabase-storage printenv | grep LIMIT\n# Should show:\n# FILE_SIZE_LIMIT=10737418240\n# UPLOAD_FILE_SIZE_LIMIT=10737418240\n```\n\n### Configuration\nEdit `/opt/supabase-project/.env` to change:\n- SMTP Settings (emails)\n- Webhook URLs\n- JWT Secrets\n\n## 📊 Resource Usage\nTypical consumption after optimization:\n- **Total RAM**: ~2.5 - 3.0 GB\n- **Disk**: ~2 GB (base installation)\n\n## 🐛 Troubleshooting\n\n**\"Realtime health check returns 404\"**\nThis is normal. The health endpoint might be hidden, but WebSocket connections will work. Test with a real client.\n\n**\"Edge Functions timeout after 60s\"**\nThis installer sets timeouts to 300s. If you see timeouts, check `volumes/api/kong.yml`.\n\n**\"Database connection failed from external IP\"**\nCheck if you ran `harden_supabase_db.sh`. If so, add your IP using option 5.\n\n## 📄 License\nMIT"
